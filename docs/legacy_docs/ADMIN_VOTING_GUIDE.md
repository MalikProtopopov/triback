# Голосование: руководство для администраторов

Документация по функционалу голосования (выборы президента/председателя ассоциации трихологов) для администраторов и менеджеров.

---

## 1. Обзор и назначение

Модуль голосования позволяет проводить выборы среди врачей-членов ассоциации. Типичный сценарий — выборы президента/председателя, членов правления или другие внутренние голосования.

**Ключевые возможности:**
- Создание сессий голосования с несколькими кандидатами
- Ограничение периода голосования по датам (`starts_at` — `ends_at`)
- Один голос на врача (дублирование невозможно)
- Просмотр результатов по кандидатам в реальном времени
- Черновик перед публикацией и ручное управление статусом

---

## 2. Жизненный цикл голосования

```
draft (черновик)
    │
    ├──► active (активно)     ◄── врачи видят голосование и могут голосовать
    │         │
    │         ├──► closed (завершено)
    │         └──► cancelled (отменено)
    │
    └──► cancelled (отменено)
```

| Этап | Описание |
|------|----------|
| **draft** | Сессия создана, но не видна врачам. Можно редактировать название, даты, список кандидатов (через PATCH). |
| **active** | Голосование открыто. Врачи видят его в ЛК и могут отдавать голос. Должно выполняться: `starts_at ≤ now < ends_at`. |
| **closed** | Голосование завершено. Результаты подсчитаны и доступны в админке. |
| **cancelled** | Голосование отменено. Результаты не считаются. |

---

## 3. Статусы и допустимые переходы

| Текущий статус | Допустимые переходы |
|----------------|---------------------|
| `draft` | `active`, `cancelled` |
| `active` | `closed`, `cancelled` |
| `closed` | — (финальный) |
| `cancelled` | — (финальный) |

**Важно:** Переход `draft` → `active` делает голосование видимым врачам. Убедитесь, что:
- `starts_at` уже наступила или близка
- `ends_at` > `starts_at`
- Кандидаты добавлены

Переход `active` → `closed` прекращает приём голосов. Дата окончания (`ends_at`) может быть скорректирована через PATCH до перехода в `closed`, чтобы продлить или сократить срок.

## 4. API-эндпоинты (полный справочник)

Базовый префикс: `/api/v1`.

Все эндпоинты требуют авторизации. Роли: **admin**, **manager** (кроме голосования врачей — только doctor).

### 4.1 Список сессий

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/admin/voting` | Пагинированный список всех сессий |

**Query-параметры:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `limit` | int | Лимит записей (по умолчанию 20) |
| `offset` | int | Смещение для пагинации |
| `status` | string | Фильтр: `draft`, `active`, `closed`, `cancelled` |

**Пример ответа:**
```json
{
  "data": [
    {
      "id": "uuid",
      "title": "Выборы председателя 2026",
      "description": "Описание голосования",
      "status": "active",
      "starts_at": "2026-04-01T09:00:00Z",
      "ends_at": "2026-04-15T18:00:00Z",
      "candidates_count": 3
    }
  ],
  "total": 5,
  "limit": 20,
  "offset": 0
}
```

    {
      "id": "uuid",

**Поля кандидата:**

- `sort_order` — порядок отображения (0, 1, 2...)

### 4.3 Создание сессии

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/admin/voting` | Создать новую сессию (статус `draft`) |

**Request body:**
```json
{
  "title": "Выборы председателя 2026",
  "description": "Описание целей голосования (опционально)",
  "starts_at": "2026-04-01T09:00:00Z",
  "ends_at": "2026-04-15T18:00:00Z",
  "candidates": [
    {
      "doctor_profile_id": "uuid-профиля-врача-1",
      "description": "Кандидат 1, опыт 15 лет..."
    },
    {
      "doctor_profile_id": "uuid-профиля-врача-2",
      "description": null
    }
  ]
}
```

**Валидация:**
- `title` — обязательно, макс. 500 символов
- `starts_at`, `ends_at` — обязательно, `ends_at` должен быть позже `starts_at`
- `candidates` — минимум 1 кандидат
- `doctor_profile_id` — должен существовать в БД

**Ответ 201:**
```json
{
  "id": "uuid",
  "title": "Выборы председателя 2026",
  "status": "draft",
  "starts_at": "2026-04-01T09:00:00Z",
  "ends_at": "2026-04-15T18:00:00Z",
  "candidates_count": 2
}
```

---

### 4.4 Обновление сессии

| Метод | Путь | Описание |
|-------|------|----------|
| PATCH | `/admin/voting/{session_id}` | Изменить название, дату окончания или статус |

**Request body (все поля опциональны):**
```json
{
  "status": "active",
  "title": "Обновлённое название",
  "ends_at": "2026-04-20T18:00:00Z"
}
```

**Ограничения:**
- `status` — только допустимый переход (см. таблицу в разделе 3)
- Нельзя изменить `starts_at` и `candidates` после создания (только через пересоздание сессии)

### 4.5 Результаты голосования

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/admin/voting/{session_id}/results` | Подсчёт голосов по кандидатам |

**Ответ:**
```json
{
  "session": {
    "id": "uuid",
    "title": "Выборы председателя 2026",
    "status": "closed",
    "total_votes": 45,
    "total_eligible_voters": 62
  },
  "results": [
    {
      "candidate": {
        "id": "uuid",
        "full_name": "Иванов Иван Петрович"
      },
      "votes_count": 28,
      "percentage": 62.22
    },
    {
      "candidate": {
        "id": "uuid",
        "full_name": "Петрова Мария Сергеевна"
      },
      "votes_count": 17,
      "percentage": 37.78
    }
  ]
}
```

**Поля:**
- `total_votes` — сколько врачей проголосовало
- `total_eligible_voters` — число врачей-членов ассоциации (статус `active` + действующая подписка на момент подсчёта)
- `results` — отсортировано по убыванию `votes_count`
- `percentage` — доля голосов от `total_votes` (округление до 2 знаков)

## 5. Как работает голосование для врачей (ЛК)

Для справки: что видят врачи и как они голосуют.

| Метод | Путь | Роль | Описание |
|-------|------|------|----------|
| GET | `/voting/active` | doctor | Текущая активная сессия (если есть) |
| POST | `/voting/{session_id}/vote` | doctor | Отдать голос за кандидата |

**Активная сессия** — та, у которой:
- `status == "active"`
- `starts_at ≤ now < ends_at`

Если таких нет, `GET /voting/active` возвращает `null`.

**Голосование:** врач отправляет `{ "candidate_id": "uuid" }`. Один голос на сессию. Повторная попытка — 409 Conflict.

**Кто может голосовать:** только активные члены ассоциации — врачи со статусом `DoctorStatus.ACTIVE` и действующей подпиской (Subscription.status = active, ends_at в будущем или null). При отсутствии подписки или неодобренном статусе — 403 Forbidden.

## 6. Бизнес-правила и ограничения

### 6.1 Кандидаты

- Кандидат — это `doctor_profile_id` (профиль врача)
- Один врач не может быть кандидатом дважды в одной сессии (уникальность)
- Порядок кандидатов задаётся при создании (порядок в массиве `candidates` → `sort_order`)

### 6.2 Голоса

- Один пользователь (`user_id`) — один голос на сессию
- Голос привязан к `candidate_id` (записи в `voting_candidates`)
- Изменить голос после отправки нельзя
- Голосовать могут только **активные члены ассоциации**: врачи с ролью `doctor`, статусом `DoctorProfile.ACTIVE` и действующей подпиской (Subscription.status = active, ends_at > now или null)

### 6.3 Даты

- В БД: `ends_at > starts_at` (CHECK constraint)
- При создании: Pydantic проверяет `ends_at > starts_at`
- Активность: голосование считается активным только при `starts_at ≤ now < ends_at`

### 6.4 Подсчёт «явки»

- `total_eligible_voters` = количество врачей с `DoctorStatus.ACTIVE` **и** действующей подпиской (status = active, ends_at > now или null)
- В подсчёт не попадают врачи с истёкшей или отсутствующей подпиской
- Явка в % = `total_votes / total_eligible_voters * 100` (можно считать на фронте)

## 7. Рекомендуемые сценарии

### Создание голосования

1. Получить список одобренных врачей: `GET /admin/doctors?status=approved`
2. Выбрать кандидатов и их `doctor_profile_id`
3. Определить даты начала и окончания
4. `POST /admin/voting` с `title`, `starts_at`, `ends_at`, `candidates`
5. Проверить черновик: `GET /admin/voting/{id}`

### Запуск голосования

1. Убедиться, что `starts_at` наступила или близка
2. `PATCH /admin/voting/{id}` с `{ "status": "active" }`
3. Оповестить врачей (через уведомления или иначе)

### Завершение голосования

**Вариант А — по времени:** после `ends_at` голосование перестаёт быть активным (никто не сможет голосовать). Результаты можно смотреть в любой момент через `GET /admin/voting/{id}/results`.

**Вариант Б — досрочно:**
1. `PATCH /admin/voting/{id}` с `{ "status": "closed" }`
2. Результаты доступны в `GET /admin/voting/{id}/results`

### Отмена

- Из `draft`: `PATCH` с `{ "status": "cancelled" }`
- Из `active`: `PATCH` с `{ "status": "cancelled" }` — голоса сохраняются в БД, но сессия больше не активна и не учитывается в отчётах как «завершённая»

## 8. Типичные ошибки и коды ответов

| Код | Ситуация |
|-----|----------|
| 401 | Не авторизован |
| 403 | Роль не admin/manager; **для POST /vote** — врач не является активным членом ассоциации (нет подписки или статус не active) |
| 404 | Сессия или кандидат не найдены |
| 409 | Врач уже проголосовал в этой сессии |
| 422 | Валидация: `ends_at <= starts_at`, недопустимый переход статуса, несуществующий `doctor_profile_id` |

**Пример 422 при недопустимом переходе:**
```json
{
  "detail": "Cannot transition from 'draft' to 'closed'"
}
```

Допустимый переход из `draft` — только в `active` или `cancelled`.

## 9. Структура данных (модели)

| Таблица | Описание |
|--------|----------|
| `voting_sessions` | Сессия: title, description, status, starts_at, ends_at, created_by |
| `voting_candidates` | Кандидаты сессии: voting_session_id, doctor_profile_id, description, sort_order |
| `votes` | Голоса: voting_session_id, user_id, candidate_id, voted_at |

**Связи:**
- Сессия → кандидаты (один ко многим)
- Сессия → голоса (один ко многим)
- Кандидат → голоса (один ко многим)

## 10. Автоматизация

**Cron-задача `close_expired_voting_sessions`** запускается каждый час и переводит сессии со статусом `active` и `ends_at < now` в статус `closed`. Ручное закрытие по-прежнему доступно через `PATCH /admin/voting/{id}` с `{ "status": "closed" }`.
