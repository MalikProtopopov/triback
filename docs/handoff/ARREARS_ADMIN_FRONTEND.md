# Админка: задолженности и оплаты врачей — что добавить и изменить

Документ для фронтенда админ-панели и бэкенд-handoff. Статус API на бэкенде: **реализовано** — сверять с OpenAPI (`/api/v1/openapi.json`).

---

## 1. Новые и изменённые API (ожидаемо)

| Область | Метод | Путь | Роли | Назначение |
|---------|-------|------|------|------------|
| Задолженности | GET | `/api/v1/admin/arrears` | admin, accountant | Список с фильтрами: `user_id`, `year`, **`status`**, `source`, **`include_inactive`** (см. ниже) |
| Задолженности | GET | `/api/v1/admin/arrears/summary` | admin, accountant | Суммы открытых/оплаченных, опц. **сумма/кол-во прощённых (waived)** |
| Задолженности | POST | `/api/v1/admin/arrears` | admin, accountant | Создать долг вручную |
| Задолженности | PATCH | `/api/v1/admin/arrears/{id}` | admin, accountant | Сумма, описание, внутренний комментарий |
| Задолженности | POST | `/api/v1/admin/arrears/{id}/cancel` | admin, accountant | Статус cancelled (ошибочное начисление и т.п.) |
| Задолженности | POST | `/api/v1/admin/arrears/{id}/mark-paid` | admin, accountant | Закрыть без провайдера (опционально связать с manual payment) |
| Задолженности | POST | `/api/v1/admin/arrears/{id}/waive` | admin, accountant | **Простить долг**: статус **waived**, аудит `waived_at`, `waived_by`, `waive_reason`; запись остаётся в БД для отчётности |
| Профиль врача | PATCH | `/api/v1/admin/doctors/{id}/payment-overrides` *(или расширение существующего PATCH)* | admin, accountant | `{ "entry_fee_exempt": true/false }` |
| Ручной платёж | POST | `/api/v1/admin/payments/manual` | admin, accountant | Расширение тела: `arrear_id` — закрыть долг при успехе |
| Дашборд | GET | `/api/v1/admin/dashboard` | admin, manager | Поля: `arrears_open_*`, `arrears_paid_*`, **`arrears_waived_total`**, **`arrears_waived_count`** (прощённые — отдельно от выручки) |
| Настройки | GET/PATCH | `/api/v1/admin/settings` | admin | Ключи: `arrears_block_membership_features`, `arrears_auto_accrual_enabled`, параметры дедлайна года (если вынесены) |

### Параметр `include_inactive` (GET `/admin/arrears`)

- **`include_inactive=true`** (по умолчанию): возвращаются все строки, согласованные с остальными фильтрами (если **`status` не задан** — полная история: open, paid, cancelled, waived).
- **`include_inactive=false`**: только строки со статусом **`open`** (очередь «к оплате»). Если одновременно передан **`status`**, он имеет приоритет: фильтр по конкретному статусу.

### Сценарий «возврат спустя годы без долгов»

- Старые требования **не удаляются** из БД: оператор переводит долг в **`waived`** (действие **«Простить долг»**).
- В ЛК врач **не видит** прощённые суммы в `open_arrears` (там только `status=open`).
- В админке запись **остаётся** в списке с **`status=waived`**, датой/кем/причиной (`waived_at`, `waived_by`, `waive_reason`) — для бухгалтерии и аудита.

## 2. Экраны и UI

### 2.1 Список задолженностей

- Таблица: врач (ФИО / email), год, сумма, **статус** (в т.ч. **«Прощено»** / `waived`), источник (ручной / авто), дата создания, кто создал (для manual); для **waived** — дата прощения, кто, краткая причина.
- Фильтры: статус (**включая waived**), год, пользователь, источник.
- Действия: создать, редактировать, отменить, отметить оплаченным вручную, **«Простить долг»** (`waive`) с обязательным/рекомендуемым полем причины.
- **Семантика:** `cancel` — отмена требования (ошибка); **`waive`** — осознанное прощение долга (возврат врача после лет без требований и т.п.); запись **не удаляется**, остаётся в списке для бухгалтерии.

### 2.2 Карточка врача (врачи админки)

- Блок **«Задолженности»**: вкладка **открытые** + вкладка/секция **«История»** (оплаченные, **прощённые**, отменённые) — чтобы видеть, какие суммы «простили», без удаления из учёта.
- Переключатель **«Не требовать вступительный взнос»** (`entry_fee_exempt`) — для миграции со старого сайта.
- Индикатор **«Исключён из ассоциации»** (если введено поле на бэкенде) — только отображение + ссылка на правила возврата.
- Кнопка **«Создать ручной платёж»** — форма ввода суммы/типа, вызов существующего manual API; при необходимости привязка `arrear_id`.

### 2.3 Дашборд

- Виджеты: **сумма открытых**, **сумма оплаченных** (через долги), **сумма и количество прощённых** (`arrears_waived_total`, `arrears_waived_count`) — отдельно от выручки; опционально число исключённых врачей (если появится в API).

### 2.4 Настройки сайта

- Тумблер **«Блокировать привилегии члена при открытых задолженностях»** (`arrears_block_membership_features`).
- Тумблер **«Автоматическое начисление задолженностей»** (`arrears_auto_accrual_enabled`) — можно скрыть до готовности backend job.
- Поля дедлайна оплаты за календарный год — если вынесены в settings.
