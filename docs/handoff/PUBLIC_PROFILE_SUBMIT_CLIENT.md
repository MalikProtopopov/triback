# Публичный профиль: `POST /api/v1/profile/public/submit` (клиентский сайт)

## Основной способ

- **Метод и путь:** `POST /api/v1/profile/public/submit`
- **Тип тела:** `multipart/form-data`
- **Заголовок:** `Authorization: Bearer <access_token>`
- **Поля формы:** любое подмножество полей публичного профиля + опционально файл **`photo`**.

Имена полей совпадают с моделью публичного профиля: `bio`, `public_email`, `public_phone`, `city_id` (UUID строкой), `clinic_name`, `academic_degree`, `specialization`, `moderation_comment`. Пустые строки в multipart **не добавляют** поле в черновик (как «поле не меняли»).

## Сценарии

1. **Только текст** — передать только изменённые текстовые поля, без `photo`.
2. **Только фото** — передать только файл в части `photo`.
3. **Всё вместе** — одна заявка `pending` с объединёнными изменениями.

Если заявка на модерацию уже есть, новые поля **дописываются в ту же строку** (вторая заявка не создаётся).

## Миграция с прежнего UX

| Раньше | Теперь |
|--------|--------|
| `PATCH /api/v1/profile/public` (JSON) для текста | Для нового потока: **`POST .../public/submit`** с теми же полями как части формы |
| `POST /api/v1/profile/photo` для аватара | Файл в **`submit`** как часть `photo` |

`PATCH /public` и `POST /photo` **сохранены** и ведут себя так же по смыслу (включая merge в существующий `pending`).

**Не относится к этому эндпоинту:** `GET/PATCH /api/v1/profile/personal`, `POST /api/v1/profile/diploma-photo` и прочие ручки вне публичной карточки.

## Фронтенд

- Собирать тело через `FormData`: для каждого изменённого поля `append("fieldName", value)`; при выборе фото — `append("photo", file)`.
- Не задавать `Content-Type` вручную (нужен boundary).
- Не добавлять в форму поля, которые пользователь не менял.

## Ответ и ошибки

- **200** — заявка принята; тело: `message`, `pending_moderation`, `photo_url` (публичный URL превью **только если в этом запросе** загрузили фото; иначе пустая строка).
- **422** — нет ни одного изменения (пустая форма без файла), невалидные данные (email, длины и т.д.).
- **404** — профиль врача не найден.

## Кратко для команды

Один запрос **`POST /api/v1/profile/public/submit`** заменяет связку «`PATCH /profile/public` + `POST /profile/photo`» для любых комбинаций текста и фото; у админа одна заявка с полным набором полей в `changes`.
