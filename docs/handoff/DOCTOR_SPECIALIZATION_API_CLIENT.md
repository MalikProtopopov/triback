# API: специализация врача (клиентский сайт и ЛК)

Специализация хранится в профиле врача как **свободный текст** (до 255 символов), без справочника и без UUID.

## Публичный каталог и карточка врача

| Метод | Путь | Поле |
|--------|------|------|
| GET | `/api/v1/doctors` | В каждом элементе `data[]`: `specialization: string \| null` |
| GET | `/api/v1/doctors/{identifier}` | `specialization: string \| null` (identifier — slug или UUID профиля) |

**Фильтр списка:** query `specialization` — подстрока без учёта регистра (`ILIKE %значение%`) по полю профиля.

## Коллеги

| Метод | Путь |
|--------|------|
| GET | `/api/v1/colleagues` |

В каждом элементе ответа: `specialization: string \| null`.

## Онбординг

| Метод | Путь | Тело |
|--------|------|------|
| PATCH | `/api/v1/onboarding/doctor-profile` | Опционально `specialization` (string, max 255) |

Поле сохраняется в профиль врача напрямую (как остальные поля анкеты).

## Личный кабинет (профиль)

| Метод | Путь | Поле `specialization` |
|--------|------|-------------------------|
| GET | `/api/v1/profile/personal` | В ответе: текущее значение с сервера |
| PATCH | `/api/v1/profile/personal` | Можно передать `specialization` — запись сразу в профиль |

## Публичный профиль (модерация)

| Метод | Путь |
|--------|------|
| GET | `/api/v1/profile/public` |
| PATCH | `/api/v1/profile/public` |

- **GET:** в корне ответа поле `specialization` — опубликованное значение из профиля.
- **PATCH:** можно отправить `specialization` в теле — уходит в **черновик** на модерацию (как `bio`, `clinic_name` и т.д.). После одобрения админом значение попадает в профиль и отображается в каталоге.

## Текущий пользователь

| Метод | Путь |
|--------|------|
| GET | `/api/v1/auth/me` |

- Для роли **`doctor`:** в ответе есть `specialization: string \| null` (из профиля врача).
- Для остальных ролей: `specialization` обычно `null`.

## Регистрация на мероприятие (гость)

Поля **`guest_specialization`** в `POST /api/v1/events/{event_id}/register` и в `confirm-guest-registration` **не связаны** с профилем врача — это отдельный произвольный текст заявки гостя (max 255). Контракт не менялся.

## Ограничения

- Длина специализации в профиле: **255 символов** (как у `guest_specialization`).
- Нормализации текста нет: разные написания = разные строки при поиске.
