# Сертификаты — Клиентский сайт

**Дата:** 2026-03-21  
**Кому:** Фронтенд-разработчик клиентского сайта  
**Тема:** Сертификаты: отображение в ЛК, скачивание, публичная верификация по QR-коду

---

## Что добавлено

1. В ЛК врача теперь отображаются **сертификаты членства** — PDF с QR-кодом, печатью и подписью президента.
2. Сертификат генерируется **автоматически** после успешной оплаты подписки.
3. Каждый сертификат содержит **QR-код**, ведущий на публичную страницу верификации.
4. Если подписка истекла — сертификат **скрывается** из ЛК.
5. Публичная страница `/certificates/verify/{number}` доступна **без авторизации** — для сканирования QR.

---

## 1. Список сертификатов: `GET /api/v1/certificates`

### Что изменилось

В ответе добавлено новое поле `verify_url`.

### Ответ

```json
[
  {
    "id": "uuid",
    "certificate_type": "member",
    "year": 2026,
    "event": null,
    "certificate_number": "TRICH-2026-000001",
    "is_active": true,
    "generated_at": "2026-03-21T12:00:00+00:00",
    "download_url": "https://s3.../presigned...",
    "verify_url": "https://trichology.ru/certificates/verify/TRICH-2026-000001"
  }
]
```

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | UUID | ID сертификата |
| `certificate_type` | string | `"member"` — членство, `"event"` — участие в мероприятии |
| `year` | int / null | Год выпуска (для member) |
| `event` | object / null | `{id, title}` — для event-сертификатов |
| `certificate_number` | string | Публичный номер сертификата |
| `is_active` | bool | Всегда `true` в этом эндпоинте (неактивные фильтруются) |
| `generated_at` | datetime | Дата генерации |
| `download_url` | string | **Presigned URL** для скачивания PDF (TTL 1 час) |
| `verify_url` | string | **Новое.** URL для верификации (ведёт на фронтенд) |

### Что делать на фронте

```js
// Раздел "Сертификаты" в ЛК
const certs = await api.get('/certificates')

// Отобразить карточки сертификатов:
// - Заголовок: "Сертификат члена ассоциации {year}"
// - Кнопка "Скачать PDF" → window.open(cert.download_url)
// - (опционально) "Поделиться" → navigator.clipboard.writeText(cert.verify_url)
```

Если массив пустой — показать заглушку:
- Если подписка active: "Сертификат генерируется, пожалуйста подождите"
- Если подписки нет: "Оплатите подписку для получения сертификата"

## 2. Скачивание: `GET /api/v1/certificates/{id}/download`

### Что изменилось

Теперь возвращает **403** если сертификат неактивен (подписка истекла).

| Код | Описание |
|-----|----------|
| 302 | Redirect на presigned S3 URL (PDF) |
| 403 | Сертификат больше неактивен |
| 404 | Не найден или принадлежит другому врачу |

На фронте: если ЛК корректно фильтрует по `is_active`, 403 возникнуть не должен. Но на всякий случай обработайте ошибку.

## 3. Публичная верификация: `GET /api/v1/public/certificates/verify/{certificate_number}`

### Новый эндпоинт (без авторизации)

QR-код на сертификате ведёт на URL вида:

```
https://trichology.ru/certificates/verify/TRICH-2026-000001
```

Фронтенд на этой странице должен вызвать:

```
GET /api/v1/public/certificates/verify/TRICH-2026-000001
```

### Ответ 200 — сертификат найден

```json
{
  "certificate_number": "TRICH-2026-000001",
  "doctor_full_name": "Маркова Юлия Алексеевна",
  "doctor_slug": "markova-yulia",
  "certificate_type": "member",
  "year": 2026,
  "issued_at": "2026-03-21T12:00:00+00:00",
  "is_valid": true,
  "invalid_reason": null,
  "organization_name": "Межрегиональная общественная организация трихологов...",
  "president_full_name": "Гаджигороева Аида Гусейхановна",
  "president_title": "Президент д.м.н."
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `certificate_number` | string | Номер сертификата |
| `doctor_full_name` | string | ФИО врача |
| `doctor_slug` | string | Slug для ссылки на профиль `/doctors/{slug}` |
| `certificate_type` | string | `"member"` или `"event"` |
| `year` | int / null | Год |
| `issued_at` | string | Дата выдачи |
| `is_valid` | bool | **Ключевое.** `true` = подписка активна; `false` = истекла |
| `invalid_reason` | string / null | Причина невалидности (если `is_valid=false`) |
| `organization_name` | string | Название организации |
| `president_full_name` | string | ФИО президента |
| `president_title` | string | Должность президента |

### Ответ 404 — сертификат не существует

Показать страницу «Сертификат не найден».

### Дизайн страницы верификации

**Если `is_valid = true`:**
- Зелёная галочка / бейдж "Действителен"
- ФИО врача (ссылка на `/doctors/{doctor_slug}`)
- Год выдачи
- Название организации
- ФИО и должность президента

**Если `is_valid = false`:**
- Красный крест / бейдж "Недействителен"
- `invalid_reason` текст (напр. "Врач более не является членом ассоциации")
- ФИО врача без ссылки

**Если 404:**
- Страница-заглушка: "Сертификат с таким номером не найден"

## 4. Новый маршрут фронта

Добавить публичный (без авторизации) роут:

```
/certificates/verify/:certificateNumber
```

Логика страницы:
1. Извлечь `certificateNumber` из URL
2. `GET /api/v1/public/certificates/verify/{certificateNumber}`
3. Отрендерить результат по дизайну выше
4. На 404 — показать заглушку
