# Правление (board_role) — Клиентский сайт

**Дата:** 2026-03-21  
**Кому:** Фронтенд-разработчик клиентского сайта  
**Тема:** Фильтрация и отображение врачей в правлении (pravlenie / president)

---

## Что добавлено

1. Поле **`board_role`** в списке и деталях врача в публичном API.
2. Фильтр по `board_role` в `GET /doctors` для отображения членов правления.

---

## API

**Префикс:** `/api/v1/public` (публичный каталог врачей)

### 1. Каталог врачей: `GET /doctors`

**Новый query-параметр:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `board_role` | list[str] | Фильтр по роли в правлении. Допустимые значения: `pravlenie`, `president`. Можно передать несколько. |

**Пример:** `GET /doctors?board_role=president` — только президент.  
`GET /doctors?board_role=pravlenie&board_role=president` — все члены правления.

В ответе каждый элемент содержит поле `board_role: string | null`.

---

### 2. Профиль врача: `GET /doctors/{slug_or_uuid}`

В ответе добавлено поле:

| Поле | Тип | Описание |
|------|-----|----------|
| `board_role` | string \| null | `"pravlenie"`, `"president"` или `null` |

---

## UI рекомендации

1. **Страница «Правление»** — запрос `GET /doctors?board_role=pravlenie&board_role=president` для списка членов правления.
2. **Разделение «Президент» / «Члены правления»** — два запроса: `?board_role=president` и `?board_role=pravlenie`, либо фильтрация на клиенте.
3. **Карточка врача** — отображать бейдж «Президент» или «Член правления» при наличии `board_role`.
