# Справочник кодов ошибок MONETA

> Выдержки из [документации docs.moneta.ru](https://docs.moneta.ru/common/dictionaries/error-codes/). Полный список — на сайте.

---

## Формат ответа

В SOAP: элемент `detail` → `faultDetail` с кодом.

```xml
<Fault>
    <faultcode>Client</faultcode>
    <faultstring>Операция не найдена</faultstring>
    <detail>
        <faultDetail>500.1.5</faultDetail>
    </detail>
</Fault>
```

В JSON:

```json
{
    "fault": {
        "detail": { "faultDetail": "500.1.5" },
        "faultcode": "Client",
        "faultstring": "Операция не найдена"
    }
}
```

## Коды, релевантные для платёжной формы Assistant

### 400 — операции и платежи

| Код | Описание |
|-----|----------|
| 400.1.5 | Оплата отменена |
| 400.1.6 | Данная операция уже оплачена |
| 400.1.18 | Сумма в заказе не совпадает с проверочной суммой |
| 400.1.19 | Операция с таким внешним идентификатором уже в обработке или выполнена |
| 400.1.22 | Выставленный счёт к оплате недействителен |

### 500.1 — не найдено

| Код | Описание |
|-----|----------|
| 500.1.5 | Операция не найдена |

## Дополнительные коды Assistant (Check URL / Pay URL)

| Код | Описание |
|-----|----------|
| 302 | Некорректный ответ на Check URL (HTML, редирект) |
| -1 | Check URL задан, но Pay URL не задан |
| -600 | Неверный Check/Pay URL или нет соединения |
| -700 | Неверный XML или подпись в ответе |

## Остальные серии (кратко)

- **100** — сервис недоступен, конфликты данных
- **200** — ошибка валидации XSD
- **300** — авторизация, доступ
- **400.2.x** — операция в очереди, ограничения по сумме
- **500.2** — неверные значения параметров
- **500.3** — детальные проверки (карта, сумма, валюта и т.д.)

Полный справочник: https://docs.moneta.ru/common/dictionaries/error-codes/
