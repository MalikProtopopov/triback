# Правление (board_role) — Админ-панель

**Дата:** 2026-03-21  
**Кому:** Фронтенд-разработчик админ-панели  
**Тема:** Роль врача в правлении (pravlenie / president)

---

## Что добавлено

1. Поле **`board_role`** в списке и деталях врача — `"pravlenie"`, `"president"` или `null`.
2. Фильтр по `board_role` в `GET /admin/doctors`.
3. Эндпоинт `PATCH /admin/doctors/{profile_id}` для обновления роли в правлении.
4. Поле `board_role` в списке и деталях пользователей портала (если это врач).

---

## API

**Префикс:** `/api/v1/admin`

### 1. Список врачей: `GET /admin/doctors`

**Роль:** admin, manager

**Новый query-параметр:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `board_role` | list[str] | Фильтр по роли в правлении. Допустимые значения: `pravlenie`, `president`. Можно передать несколько значений. |

**Пример:** `GET /admin/doctors?board_role=pravlenie&board_role=president`

В ответе каждый элемент списка содержит поле `board_role: string | null`.

---

### 2. Детали врача: `GET /admin/doctors/{profile_id}`

**Роль:** admin, manager

В ответе добавлено поле:

| Поле | Тип | Описание |
|------|-----|----------|
| `board_role` | string \| null | `"pravlenie"`, `"president"` или `null` |

---

### 3. Обновить роль в правлении: `PATCH /admin/doctors/{profile_id}`

**Роль:** admin, manager

**Тело запроса:**

```json
{
  "board_role": "pravlenie"
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `board_role` | `"pravlenie"` \| `"president"` \| null | Роль в правлении. `null` — убрать роль. |

**Ответ (200):** полный объект `DoctorDetailResponse` с обновлённым `board_role`.

**Ошибки:**
- **404** — профиль не найден
- **401** — не авторизован
- **403** — роль не admin/manager

---

### 4. Пользователи портала

В `GET /admin/portal-users` и `GET /admin/portal-users/{user_id}` каждый элемент (для врачей) содержит поле `board_role: string | null`.

---

## UI рекомендации

1. **Список врачей** — добавить фильтр «Роль в правлении» (dropdown/checkboxes: Правление, Президент, Без роли).
2. **Карточка врача** — поле «Роль в правлении» с возможностью редактирования (select: — / Правление / Президент), сохранять через `PATCH`.
3. **Список пользователей портала** — отображать `board_role` в карточке врача (например, бейдж).
