# Голосование — Клиентский сайт (ЛК)

**Дата:** 2026-03-23  
**Кому:** Фронтенд-разработчик клиентского сайта  
**Тема:** Обновление голосования — проверка активного члена ассоциации

---

## Что изменилось в бэкенде

1. **Проверка при голосовании** — голосовать могут только **активные члены ассоциации** (DoctorProfile.status = ACTIVE + действующая подписка).
2. **403 Forbidden** — при отсутствии подписки или неодобренном статусе `POST /voting/{id}/vote` возвращает 403.
3. **Сообщение об ошибке:** `"Голосовать могут только активные члены ассоциации"`.

---

## API

**Эндпоинт без изменений:** `GET /api/v1/voting/active`, `POST /api/v1/voting/{session_id}/vote`

### Ошибки POST `/voting/{session_id}/vote`

| Код | Когда | Сообщение |
|-----|-------|-----------|
| 403 | Врач не активный член (нет подписки / статус не ACTIVE) | «Голосовать могут только активные члены ассоциации» |
| 404 | Сессия или кандидат не найдены | — |
| 409 | Уже проголосовал | ConflictError |

---

## Рекомендации для фронтенда

1. **Обработка 403 при голосовании:**
   - Показать toast/alert с текстом из `detail`.
   - Добавить ссылку на продление подписки: `/cabinet/subscription` или `/cabinet/payments`.

2. **Опционально (улучшение UX):**
   - Перед отображением кнопки «Проголосовать» проверять `GET /subscriptions/status`.
   - Если подписка неактивна — показывать информационный блок: «Голосовать могут только активные члены ассоциации. [Продлить подписку](/cabinet/subscription)».
   - Либо дизейблить кнопку и показывать tooltip.

3. **Сценарий:** врач с истёкшей подпиской видит активное голосование (GET /voting/active возвращает данные), нажимает «Проголосовать» → 403 → показать сообщение + ссылку.

---

## Пример обработки ошибки

```typescript
// При POST /voting/{id}/vote
if (response.status === 403) {
  const message = response.detail || "Голосовать могут только активные члены ассоциации";
  showToast(message);
  // Опционально: показать кнопку «Продлить подписку» → navigate("/cabinet/subscription")
}
```
