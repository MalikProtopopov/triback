# Клиентский сайт / ЛК врача: задолженности и оплата — что сделать

Документ для фронтенда личного кабинета врача. Статус API на бэкенде: **реализовано** — сверять с OpenAPI (`/api/v1/openapi.json`).

---

## 1. Единый экран «Оплата членства»

### 1.1 Источник данных

- **Один запрос** (рекомендуется): `GET /api/v1/subscriptions/status` — расширенный ответ.
- В ответе ожидаются (концептуально):
  - текущая подписка, `available_plans`, `entry_fee_plan`, `entry_fee_required`, `next_action`;
  - **`open_arrears`** — только **`status=open`** (к оплате). Задолженности в статусе **«прощено»** (`waived` на бэкенде) **сюда не попадают** — врач их не видит как долг; прощение — только учёт в админке.
  - **`arrears_total`**;
  - **`arrears_block_active`** — если `true` и задана настройка блокировки, UI показывает предупреждение (скрытие в каталоге, отсутствие скидок — по продукту);
  - **`is_membership_excluded`**, **`membership_excluded_at`** — для сценария «исключён / возврат»;
  - у каждого элемента **`open_arrears`**: при наличии автоматической логики в будущем — опционально **`escalation_level`** (бейдж срочности; сейчас может быть `null`).

Отдельный запрос только за долгами **не обязателен**; опционально `GET /api/v1/subscriptions/payments` (или существующий список платежей) для истории.

**Прощённые долги (`waived`):** врач **никогда** не получает их в `open_arrears` — только открытые требования к оплате.

---

## 2. Отображение задолженностей

- Блок **«Задолженности»** над или под списком планов: таблица/карточки по строкам (год, сумма, текст для врача; опц. `escalation_level`).
- Кнопка **«Оплатить»** у каждой открытой строки (если оплата по одному долгу за раз).

---

## 3. Оплата задолженности

- **POST** `/api/v1/subscriptions/pay-arrears` (или финальный путь из OpenAPI).
- Тело: `{ "arrear_id": "<uuid>", "idempotency_key": "<string>" }`.
- Ответ: как у обычной оплаты — `payment_url`, `amount`, `expires_at` → редирект на Moneta, обработка возврата и ошибок.

---

## 4. Очередь оплаты (продукт)

- Если бэкенд требует **сначала** погасить долги, **потом** купить членский за текущий год — UI:
  - отключить кнопку «Оплатить членский» пока есть `open_arrears`, **или**
  - показывать одну агрегированную оплату (если бэкенд позже добавит объединённый чек).

Текущий план бэкенда: уточнить в handoff после реализации; до этого следовать полям `next_action` и сообщениям валидации API.

---

## 5. История платежей

- В списке платежей врача отображать типы, в т.ч. `membership_arrears`, с человекочитаемым названием: «Задолженность за {год}».
- Источник: поля `description`, `product_type`, при необходимости связь с `arrear_id` в ответе API.

---

## 6. Каталог на публичном сайте

- При `arrears_block_active === true` и открытых долгах врач может **не отображаться** в каталоге врачей (правило на бэкенде).
- В ЛК показать понятное сообщение: «Погасите задолженность, чтобы восстановить отображение профиля» (текст согласовать с юридическим/продуктом).

---

## 7. Сценарий «исключён после пропусков» и возврат

- Если врач в статусе **исключён**:
  - показать блок **«Возврат в ассоциацию»**: вступительный по текущему тарифу + членский за первый год + список непогашенных долгов за прошлые годы.
- Очередь и кнопки — по ответу `GET /subscriptions/status` и правилам `next_action` от бэкенда.

**Возврат после долгого перерыва без долгов перед врачом:** если бухгалтерия **простила** старые долги (`waived`), в ЛК **не показывать** эти суммы — в `open_arrears` их нет. Врач оплачивает только актуальные требования (вступительный/членский по правилам бэкенда). Историю прощений врачу **не** выводить, если нет продуктовой необходимости.

---

## 8. Чек-лист для фронта

- [ ] Загрузка одного статуса подписки с долгами и флагами.
- [ ] Блок **только открытых** задолженностей (`open_arrears`) + оплата по `arrear_id` — без отображения **waived** (прощённые админом не в этом списке).
- [ ] Учёт `entry_fee_exempt` (не показывать вступительный, если бэкенд так отдал).
- [ ] История платежей с типом задолженности (успешные оплаты долгов).
- [ ] Сообщения при блокировке каталога / скидок.
- [ ] Экран «возврат» для исключённых (когда API готово).
- [ ] Не ожидать в API список прощённых долгов для врача — только открытые к оплате.
