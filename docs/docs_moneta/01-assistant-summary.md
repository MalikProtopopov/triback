## Общее описание

MONETA.Assistant — API для приёма платежей через разные платёжные системы (МОНЕТА.РУ, WebMoney, Яндекс.Деньги и др.). Магазин интегрирует только MONETA.Assistant, а покупатель может платить любой подключённой системой.

### Процесс оплаты (без счёта в МОНЕТА.РУ)

1. Покупатель выбирает товар.
2. Магазин формирует заказ и реквизиты.
3. Покупатель видит форму MONETA.Assistant и выбирает способ оплаты.
4. МОНЕТА.РУ формирует форму оплаты.
5. Покупатель подтверждает оплату.
6. Внешняя платёжная система уведомляет МОНЕТА.РУ.
7. МОНЕТА.РУ отправляет магазину отчёт об операции.
8. МОНЕТА.РУ предлагает покупателю вернуться на сайт магазина.

### Что нужно на стороне магазина

- Форма с параметрами заказа (POST на `https://moneta.ru/assistant.htm`).
- Обработчик Pay URL для приёма уведомлений об оплате.
- При необходимости — Check URL для проверочных запросов.

## С чего начать

### Demo‑площадка

- **URL:** https://demo.moneta.ru  
- Счета (MNT_ID) на demo и боевой платформе разные.

### Регистрация на demo

1. Создать личный кабинет магазина: https://demo.moneta.ru/backoffice/auth/register  
2. Создать личный кабинет покупателя (другой email): https://demo.moneta.ru/locale.htm?moneta.locale=ru&redirect=/register.htm  
3. Написать в поддержку PayAnyWay (business@support.payanyway.ru) для активации ЛК магазина и пополнения виртуальными деньгами.  
4. В ЛК магазина: `Мой счет > Управление счетами > Расширенные счета > Добавить счет` — создать расширенный счёт.

### Настройки счёта магазина

Раздел: `Мой счет > Управление счетами`.

| Параметр | Описание |
|----------|----------|
| **Тестовый режим** | Движения средств не происходят, только обмен данными. Режим задаётся только этим флагом; `MNT_TEST_MODE` устарел. |
| **Список платёжных систем** | Список ID через запятую (например, 1015,1017). Пусто — все доступные. Можно передать `paymentSystem.limitIds` в запросе. |
| **Check URL** | URL обработчика проверочных запросов. |
| **Pay URL** | URL обработчика уведомлений об оплате. |
| **HTTP Method** | GET или POST для Check URL и Pay URL. |
| **Код проверки целостности данных** | Секрет для подписи (известен только МОНЕТА.РУ и магазину). |
| **Подпись формы обязательна** | Обязательность параметра `MNT_SIGNATURE` в запросе. Не включать, если не передаётся `MNT_TRANSACTION_ID`. |
| **Можно переопределять настройки в URL** | Разрешение задавать Success/Fail/InProgress/Return URL в параметрах запроса. |
| **Success URL** | Страница после успешной оплаты. Добавляется `MNT_TRANSACTION_ID`. |
| **Fail URL** | Страница после отмены оплаты. `MNT_TRANSACTION_ID` добавляется. |
| **InProgress URL** | Страница до подтверждения списания (для части методов). |
| **Return URL** | Страница при добровольном отказе от оплаты. |

## Версии платёжной формы (v1, v2, v3)

Существует три варианта платёжной формы. Backend (Check URL, Pay URL) единый для всех версий.

| Версия | Описание | Способы оплаты |
|--------|----------|----------------|
| **v1** | Классическая форма (HTML POST на assistant.htm или iframe assistant.widget) | МОНЕТА.РУ, WebMoney, Яндекс.Деньги, карты и др. |
| **v2** | Компактная форма (JS `Assistant.Builder`) | Только банковские карты |
| **v3** | Новая форма | Карты (российские и иностранные), СБП, SberPay, рекуррентные платежи |

### Включение v3

**Через ЛК:** `Счета` → выбрать счёт → `Настроить способы оплаты` → включить новую версию.

**Через запрос:** передать параметр `version=v3` в запросе на оплату.

## Запрос на оплату (Payment Request)

Форма подготавливается магазином, подтверждается пользователем и отправляется в МОНЕТА.РУ.

### Endpoints

| Среда | URL |
|-------|-----|
| Продакшн | `https://moneta.ru/assistant.htm` |
| Demo | `https://demo.moneta.ru/assistant.htm` |

### Минимальная ссылка на оплату

Для формирования ссылки на виджет достаточно двух параметров: `MNT_ID` и `MNT_AMOUNT`:

```
https://moneta.ru/assistant.widget?MNT_ID=НОМЕР_СЧЁТА&MNT_AMOUNT=СУММА_ПЛАТЕЖА
```

Ссылку можно передать пользователю или использовать в iframe. Остальные параметры (MNT_TRANSACTION_ID, Success/Fail URL и т.д.) опциональны, но рекомендуются для корректной интеграции.

### Альтернатива: MerchantAPI → operationId

Если инвойс создаётся через MerchantAPI (метод `InvoiceRequest`), в ответе в поле `transaction` приходит номер операции. Перенаправить покупателя на оплату можно по ссылке:

```
https://moneta.ru/assistant.widget?operationId=НОМЕР_ОПЕРАЦИИ
```

### Параметры формы (POST)

**Обязательные:**

| Параметр | Описание |
|----------|----------|
| `MNT_ID` | Номер счёта магазина. |
| `MNT_TRANSACTION_ID` | Уникальный ID заказа в магазине. |
| `MNT_AMOUNT` | Сумма платежа. |
| `MNT_CURRENCY_CODE` | Код валюты (RUB и т.п.). |

**Рекомендуемые / часто используемые:**

| Параметр | Описание |
|----------|----------|
| `MNT_SUCCESS_URL` | URL после успешной оплаты. |
| `MNT_FAIL_URL` | URL после неуспешной оплаты. |
| `MNT_RETURN_URL` | URL при отказе от оплаты. |
| `MNT_SIGNATURE` | Подпись (если в настройках включена обязательная подпись). |

**Дополнительные:**

| Параметр | Описание |
|----------|----------|
| `followup` | `true`/`false` — автоматическое прохождение Assistant. |
| `javascriptEnabled` | `true`/`false` — поддержка JS в браузере. |
| `paymentSystem.accountId` | ID платёжной системы (например, WebMoney WMR = 2). |
| `paymentSystem.limitIds` | Ограничение списка платёжных систем (через запятую). |
| `customParams` | Пользовательские параметры. |

### Пример HTML‑формы

```html
<form method="post" action="https://moneta.ru/assistant.htm">
  <input type="hidden" name="MNT_ID" value="00000001">
  <input type="hidden" name="MNT_TRANSACTION_ID" value="FF790ABCD">
  <input type="hidden" name="MNT_CURRENCY_CODE" value="RUB">
  <input type="hidden" name="MNT_AMOUNT" value="120.25">
  <input type="submit" value="Оплатить">
</form>
```

### iframe

- Минимальная ширина: 320px, желательно 488px.  
- Высота зависит от числа методов оплаты.

## Уведомление о проведённой оплате (Pay URL)

МОНЕТА.РУ отправляет на Pay URL HTTP GET или POST с параметрами операции. Магазин должен вернуть ответ, подтверждающий получение.

### Параметры уведомления

| Параметр | Описание |
|----------|----------|
| `MNT_ID` | Номер счёта. |
| `MNT_TRANSACTION_ID` | ID заказа в магазине. |
| `MNT_OPERATION_ID` | ID операции в МОНЕТА.РУ. |
| `MNT_AMOUNT` | Сумма. |
| `MNT_CURRENCY_CODE` | Код валюты. |
| `MNT_TEST_MODE` | Тестовый режим (0/1). |
| `MNT_SIGNATURE` | Подпись для проверки подлинности. |

### Проверка подписи входящего запроса

Подпись MNT_SIGNATURE формируется как **MD5** от конкатенации значений параметров (по одним источникам — в алфавитном порядке ключей) и кода проверки целостности. **Точный порядок полей и список участвующих параметров см. в [PDF‑спецификации MONETA.Assistant](https://www.moneta.ru/doc/MONETA.Assistant.ru.pdf)** — в веб‑документации он не раскрыт полностью.

### Требования к ответу

- Кодировка: UTF-8.  
- Формат: XML или текст (без HTML).

**Текстовый успешный ответ:**
```
SUCCESS
```

**XML‑ответ:**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<MNT_RESPONSE>
  <MNT_ID>11223344</MNT_ID>
  <MNT_TRANSACTION_ID>FF790ABCD</MNT_TRANSACTION_ID>
  <MNT_RESULT_CODE>200</MNT_RESULT_CODE>
  <MNT_SIGNATURE>29807c8e5d82198b5c4360e6ec711cce</MNT_SIGNATURE>
</MNT_RESPONSE>
```

### Формула подписи ответа (MNT_SIGNATURE)

```
MNT_SIGNATURE = md5(MNT_ID + MNT_TRANSACTION_ID + MNT_RESULT_CODE + <код_проверки_целостности>)
```

Или в общем виде:
```
MNT_SIGNATURE = md5(<параметры_в_порядке_согласно_документации> + <секрет>)
```

Точный порядок полей см. в [PDF‑документации](https://www.moneta.ru/doc/MONETA.Assistant.ru.pdf).

### Поведение при отсутствии ответа

- Повторные попытки ~26–27 часов.  
- Если ответа нет — операция считается выполненной без подтверждения магазином.

### Когда повторяют отправку

- Текст ответа начинается с `FAIL`.  
- Сервер магазина недоступен.  
- Ответ от обработчика не получен.

### Идемпотентность

При повторных уведомлениях по той же операции нужно снова возвращать успешный ответ (`SUCCESS` или XML с кодом 200), чтобы остановить повторные попытки.

## Проверочные запросы (Check URL)

Если в настройках счёта задан Check URL, магазин получает проверочные запросы **до** или **во время** оплаты.

### Назначение

- Проверка, что заказ существует и ещё не оплачен.
- Уточнение суммы, если она не была передана в форме.
- Проверка срока действия заказа.
- Получение данных, зависящих от введённых параметров (step checks).

### Формат ответа

Обязательно XML. Поля:

| Поле | Описание |
|------|----------|
| `MNT_ID` | ID счёта. |
| `MNT_TRANSACTION_ID` | ID транзакции. |
| `MNT_RESULT_CODE` | Код результата (200 — успех). |
| `MNT_AMOUNT` | Сумма (если нужно передать/уточнить). |
| `MNT_DESCRIPTION` | Описание (опционально). |
| `MNT_SIGNATURE` | Подпись ответа. |

### Обработка прекращается, если

- `MNT_SIGNATURE` не совпадает с ожидаемым.
- `MNT_RESULT_CODE` не из списка допустимых.
- `MNT_ID` или `MNT_TRANSACTION_ID` не совпадают с запросом.
- Ответ не в формате XML.
- Ответ не получен от обработчика.

## Автоматическое прохождение Assistant

Позволяет пропустить шаги выбора платёжной системы и ввода параметров, если они известны заранее.

### Параметры

| Параметр | Описание |
|----------|----------|
| `followup` | `true` — автоматический проход по шагам; `false` — ручной ввод. По умолчанию `false`. |
| `javascriptEnabled` | `true`/`false` — поддержка JS. Важно передавать реальное значение. |
| `paymentSystem.accountId` | ID счёта/типа платёжной системы (например, 2 — WebMoney WMR, 3 — WMZ). |

### Пример формы (WebMoney WMR)

```html
<form method="post" action="https://moneta.ru/assistant.htm">
  <input type="hidden" name="MNT_ID" value="00000001">
  <input type="hidden" name="MNT_TRANSACTION_ID" value="FF790ABCD">
  <input type="hidden" name="MNT_CURRENCY_CODE" value="RUB">
  <input type="hidden" name="MNT_AMOUNT" value="120.25">
  <input type="hidden" name="paymentSystem.accountId" value="2">
  <input type="hidden" name="javascriptEnabled" value="true">
  <input type="hidden" name="followup" value="true">
  <input type="submit" value="Pay with Webmoney WMR">
</form>
```

## Получение данных от поставщика (Step Checks)

Дополнительные Check URL‑запросы после каждого шага ввода данных (например, лицевой счёт, тип кошелька и т.п.).

### Применение

- Указание суммы, зависящей от введённых параметров.
- Получение данных от поставщика (адрес, показания и т.д.) по уже введённым данным.

Правила ответа такие же, как для обычного Check URL: только XML, проверка подписи и кодов.

## Виджет

Виджет — форма оплаты, встроенная в страницу магазина (в корзину и т.п.).

### URL виджета

```
https://moneta.ru/assistant.widget?MNT_ID=...&MNT_CURRENCY_CODE=RUB&MNT_AMOUNT=...
```

### Пример iframe

```html
<iframe src="https://moneta.ru/assistant.widget?MNT_ID=19312235&MNT_CURRENCY_CODE=RUB&MNT_AMOUNT=10.25">
  Ваш браузер не поддерживает iframe!
</iframe>
```

### Особенности

- Success/Fail URL открываются внутри iframe — страницы должны корректно отображаться во фрейме.
- При изменении корзины без перезагрузки нужно обновлять параметры iframe (в т.ч. сумму).
- Оформление можно менять через CSS (см. [документацию по кастомизации](https://docs.moneta.ru/assistant/v1/customization/)).

## Настройка виджета (Assistant.Builder)

При использовании JS‑виджета (`https://payanyway.ru/assistant-builder`) Check URL и Pay URL работают так же. Параметры из формы передаются в запросах на эти endpoints.

### Параметры options (JavaScript)

| Параметр | Описание |
|----------|----------|
| `account` | Номер бизнес‑счёта. |
| `amount` | Сумма платежа. |
| `transactionId` | Номер заказа в магазине. |
| `operationId` | ID операции в Монете (опц.). |
| `description` | Описание заказа (опц.). |
| `signature` | Подпись запроса (опц.). |
| `subscriberId` | ID клиента в магазине (опц.). |
| `testMode` | `0` — боевой, `1` — тестовый (по умолчанию 0). |
| `lang` | `ru` / `en` (по умолчанию `ru`). |
| `theme` | `light` / `dark` (по умолчанию `light`). |
| `customParams` | Объект с произвольными полями — все попадают в Check и Pay URL. |

### Callbacks

```javascript
assistant.setOnSuccessCallback(function(operationId, transactionId) { ... });
assistant.setOnFailCallback(function(operationId, transactionId) { ... });
assistant.setOnInProgressCallback(function(operationId, transactionId) { ... });
```

## Дополнительные коды ошибок

| Код | Описание | Рекомендация |
|-----|----------|--------------|
| **302** | Некорректный ответ на Check URL (HTML, редирект 301/302 и т.п.). | Убедиться, что Check URL возвращает корректный XML. |
| **-1** | Запроса не было. В настройках задан Check URL, но не задан Pay URL. | Настроить Pay URL. |
| **-600** | Не удалось отправить уведомление. Неверный Check/Pay URL или отсутствие соединения. | Проверить URL, доступность с внешних сетей (не localhost и т.п.). |
| **-700** | Некорректный XML или подпись в ответе. | Проверить формат XML и алгоритм подписи, совпадение кода проверки целостности. |

## Справочники

### Статусы операций

| Статус | Описание |
|--------|----------|
| `CREATED` | Операция создана. |
| `INPROGRESS` | Обработка или ожидание взаимодействия с плательщиком. |
| `TAKENOUT` | Средства списаны. |
| `TAKENIN_NOTSENT` | Зачислено, уведомление ещё не отправлено. |
| `FROZEN` | Ручной разбор администраторами. |
| `SUCCEED` | Успешно завершено. |
| `CANCELED` | Отменено (с возвратом, если были зачисления/списания). |

### Список валют

| Код | Название |
|-----|----------|
| RUB | Российский рубль |
| USD | Доллар США |
| EUR | Евро |
| GBP | Фунт стерлингов |
| CNY | Юань |

### Виды операций

| Тип | Описание |
|-----|----------|
| **Оплата** | Деньги со счёта держателя в пользу ТСП. Можно отменить (только при двухстадийной оплате). |
| **Возврат** | ТСП возвращает средства держателю. Связан с операцией оплаты. Срок зачисления — до 3 дней. |
| **Отмена** | Разблокировка средств на карте при ошибочной оплате (только при холдировании). |

Полный справочник кодов ошибок см. в [MONETA_Error_Codes_Reference.md](MONETA_Error_Codes_Reference.md).

Коды временных зон: [docs.moneta.ru/common/dictionaries/timezones](https://docs.moneta.ru/common/dictionaries/timezones/) (для 54‑ФЗ, чеков).

## Термины и определения

| Термин | Описание |
|--------|----------|
| **Монета** | Платёжная система (НКО «МОНЕТА»). |
